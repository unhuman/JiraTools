## Importing Tickets via Claude Code MCP

As an alternative to the `--csv` import workflow, you can use Claude Code with a Jira MCP server to create tickets directly from the generated CSV files.

### Prerequisites

- [Claude Code](https://claude.ai/code) CLI installed
- A Jira MCP server configured in Claude Code (e.g. `jira-mcp`)
- The per-team CSV files generated by `standardTicketCreator.py --csv`
- Your Excel config file (e.g. `teams.xlsx`) with the sheets described below

### Excel File Requirements

Your Excel config file must include a **`CustomFields`** sheet that maps logical field names to Jira custom field IDs:

| Key | Value | Data Wrapper |
|-----|-------|-------------|
| Sprint Team | `customfield_XXXXX` | `value` |
| Epic Link | `customfield_XXXXX` | `none` |
| Sprint | `customfield_XXXXX` | `none` |

> **Note:** You can discover your Jira instance's custom field IDs using `findCustomFields.py`.

### Field Mapping Notes

When instructing Claude Code to create tickets, the following field mapping rules apply:

- **Epic Link** — use the `epicKey` additional field (e.g. `{"epicKey": "PROJ-123"}`), **not** the raw `customfield_XXXXX` value. Using the custom field directly may result in a permissions error even when you have access to the epic.
- **Sprint Team** — pass as `{"customfield_XXXXX": {"value": "TeamName"}}` (requires the `value` wrapper per the `Data Wrapper` column).
- **Sprint** — pass as `{"customfield_XXXXX": <sprint_id>}` where the sprint ID is the numeric value from the Teams sheet.
- **Priority** — pass as `{"priority": {"name": "4-Medium"}}` (standard Jira field, not a custom field).
- Use `jira_create_issue` for each ticket individually — `jira_batch_create_issues` does not support `additional_fields`, so Priority, Epic Link, Sprint, and Sprint Team would be silently dropped.

### Process

1. Generate the per-team CSV files:
   ```bash
   python standardTicketCreator.py teams.xlsx --csv output.csv
   ```
2. Open Claude Code and ensure the Jira MCP server is active.
3. Instruct Claude Code to read your Excel config file and the CSV files, then create tickets using `jira_create_issue` with the field mappings above.
4. Claude Code can process all teams in parallel, creating one ticket per CSV row.
5. Review the ticket keys returned and verify Epic Link attachment in Jira (the `epicKey` field is accepted but may not always appear in the API response).
