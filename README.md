Useful Stuff for working with Jira

### Importing Tickets from CSV

Use `createTicketsFromCsv.py` to create Jira tickets from the per-team CSV files generated by `standardTicketCreator.py --csv`. This approach is deterministic and avoids the manual error-prone loop required when using Claude Code.

### Prerequisites

- The per-team CSV files generated by `standardTicketCreator.py --csv` (e.g., `q3-2026-Queueless.csv`)
- Your Excel config file (e.g. `teams.xlsx`) with a **`CustomFields`** sheet that maps logical field names to Jira custom field IDs:

| Key | Value | Data Wrapper |
|-----|-------|-------------|
| Sprint Team | `customfield_XXXXX` | `value` |
| Epic Link | `customfield_XXXXX` | `none` |
| Sprint | `customfield_XXXXX` | `none` |

> **Note:** You can discover your Jira instance's custom field IDs using `findCustomFields.py`.

### Process

1. Generate the per-team CSV files:
   ```bash
   python standardTicketCreator.py teams.xlsx --csv output.csv
   ```

2. (Optional) Verify what would be created without calling Jira:
   ```bash
   python createTicketsFromCsv.py q3-2026-Queueless.csv teams.xlsx --dry-run
   ```

3. Create the tickets:
   ```bash
   python createTicketsFromCsv.py q3-2026-Queueless.csv teams.xlsx
   ```

   Repeat for each team's CSV file, or wrap in a loop:
   ```bash
   for csv in q3-2026-*.csv; do
     python createTicketsFromCsv.py "$csv" teams.xlsx
   done
   ```

4. Review the printed ticket keys to confirm all were created successfully.

### Alternative: Manual Creation via Claude Code

If you prefer to use Claude Code with the Jira MCP, follow these field mapping rules:

- **Description** — Always use the **complete, full-length text** from the CSV `Description` column. Do not abbreviate, summarize, or truncate descriptions. The description field often contains detailed improvement opportunities, metrics, and structured data that teams rely on. Truncation causes loss of critical information and requires post-creation updates.
- **Epic Link** — use the `epicKey` additional field (e.g. `{"epicKey": "PROJ-123"}`), **not** the raw `customfield_XXXXX` value.
- **Summary** — Pass the exact `Summary` value from each CSV row; do **not** reuse the summary from the first row for subsequent rows.
- Use `jira_create_issue` for each ticket individually — `jira_batch_create_issues` does not support `additional_fields`, so Priority, Epic Link, Sprint, and Sprint Team would be silently dropped.
